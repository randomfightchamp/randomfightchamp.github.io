# Случайные Соревнования

Визуальный рандомайзер для определения победителя среди 2–100 участников через зрелищные мини-игры на Canvas. Интерактивное веб-приложение для определения победителя среди N участников через серию случайных мини-игр. Приложение работает как единая HTML-страница, размещаемая на GitHub Pages.

**Сайт:** [randomfightchamp.github.io](https://randomfightchamp.github.io)

## Быстрый старт

1. Откройте `index.html` в браузере (или запустите локальный сервер).
2. Настройте команду участников.
3. Выберите одну из пяти игр.
4. На арене нажмите `Старт` и наблюдайте, кто останется последним.

# 🎮 Случайные Соревнования

Интерактивное веб-приложение для определения победителя среди N участников через серию случайных мини-игр. Приложение работает как единая HTML-страница, размещаемая на GitHub Pages.

## 📋 Основные особенности

- **Темная тема** с неоновыми акцентами и анимациями
- **5 уникальных мини-игр** с различными механиками
- **Поддержка до 100 участников** с возможностью редактирования имен и цветов
- **Система эскалации** - игры усложняются при долгом отсутствии выбываний
- **Адаптивный дизайн** для современных браузеров (Chrome, Firefox, Edge)

## 🕹️ Доступные игры

### 1. 🧊 Ледяное Сумо

**Описание:** Круглая арена. Участники скользят по льду, получая случайные импульсы. Столкновения наносят урон. Последний выживший — победитель.

**Механика:**
- Участники имеют 3 HP, при получении урона становятся временно неуязвимыми
- Столкновения передают импульсы между участниками
- При эскалации арена уменьшается, ускоряя игру
- Участники получают случайные толчки каждые 500мс

### 2. 🎯 Арканоид Судьбы

**Описание:** Два вложенных многоугольника, вращающихся в противоположных направлениях. Внутри летает шарик, который разрушает внутренний многоугольник и выбивает участников из внешнего.

**Механика:**
- Внутренний многоугольник (30 граней) разрушается при ударах шарика
- Внешний многоугольник (по числу участников) содержит игроков
- Шарик, вылетев из внутреннего, попадает в внешний многоугольник - выбывает игрок
- При финале (2 участника) внешний многоугольник становится квадратом
- При эскалации разрушаются грани внутреннего многоугольника, увеличивается скорость шарика

### 3. 🧬 Клеточный Делёж

**Описание:** Тёмное поле с плавающими цветными клетками. Клетки едят пищу, растут, делятся и сталкиваются. Большая клетка наносит урон меньшей. Побеждает последняя выжившая.

**Механика:**
- Клетки едят еду для роста, при достижении макс. размера делятся
- Крупные клетки наносят урон мелким при столкновении
- У каждой клетки может быть "спутник" - дополнительная защита
- При эскалации вся еда становится ядовитой, уменьшая здоровье и размер
- Крупные клетки получают эффект "дрожания" для визуального эффекта

### 4. 🌀 Сужающийся Шторм (в разработке)

**Описание:** Область безопасной зоны постепенно сужается, участники должны уклоняться от снарядов.

**Механика:**
- Безопасная зона постепенно уменьшается
- Появляются снаряды, летящие из внешних областей
- При эскалации увеличивается скорость сужения и количество снарядов

### 5. 🦠 Микробная Колония (в разработке)

**Описание:** Участники контролируют микробы на сетке, захватывают территории и расширяют свои колонии.

**Механика:**
- Игра на сетке, каждый участник захватывает территории
- Конфликты между колониями решаются по размеру территории
- При эскалации ускоряется захват территорий

## 🛠️ Технические особенности

- **Стек:** HTML5, CSS3, Vanilla JavaScript (ES6+)
- **Без внешних зависимостей** - все на нативном JavaScript
- **Canvas API** для рендеринга игровых сцен
- **Трехэкранная система:** Сбор команды → Выбор игры → Арена
- **Система кулдаунов:** Минимум 10 секунд между выбываниями для лучшего наблюдения
- **Система эскалации:** Если 30 секунд нет выбываний, игра усложняется

## 🎨 Визуальные эффекты

- **Неоновые цвета** (#00ffcc, #ff00aa, #ffcc00)
- **Анимации** плавного появления/исчезновения
- **Частицы** при выбывании участников
- **Тряска камеры** при эскалации
- **Свечение и градиенты** для улучшения визуального восприятия

## 📱 Управление

1. **Сбор команды:** Установите количество участников, отредактируйте имена и цвета
2. **Выбор игры:** Выберите одну из доступных мини-игр
3. **Наблюдение:** Наблюдайте за соревнованием, пока не останется один победитель
4. **Повтор:** Используйте кнопку "Сыграть ещё раз" для новой игры

## 🔧 Установка и запуск

1. Клонируйте репозиторий
2. Откройте `index.html` в браузере через локальный сервер (например, Live Server в VS Code)
3. Приложение готово к использованию

## 🏆 Цель игры

Определить одного победителя среди N участников через серию случайных испытаний, где каждый раунд уменьшает количество участников до тех пор, пока не останется только один.
