# Случайные Соревнования

Визуальный рандомайзер для определения победителя среди участников через серию мини-игр на HTML5 Canvas. Проект размещается как статический сайт на GitHub Pages (`randomfightchamp.github.io`).

## Назначение

Приложение помогает честно и зрелищно выбрать одного победителя из группы участников (2–100 человек). Участники не управляют персонажами вручную — всё происходит автоматически на основе физики, случайности и внутриигровых правил. Итог фиксируется, когда остаётся один выживший.

## Как пользоваться

1. **Сбор команды** — задайте количество участников (2–100), при необходимости отредактируйте имена и цвета.
2. **Выбор игры** — выберите одну из пяти мини-игр.
3. **Арена** — проверьте стартовую расстановку и нажмите **Старт**.
4. **Победитель** — по окончании игры появляется модальное окно с именем победителя.

### Элементы управления на арене

| Кнопка | Действие |
|--------|----------|
| **Старт** | Запускает выбранную игру после открытия арены |
| **Остановить / Рестарт** | Прерывает текущую игру и возвращает к выбору игр |
| **🔊 / 🔇** | Переключение звука (визуальная кнопка; звуки генерируются через Web Audio API) |

## Общие правила для всех игр

### Участники

- Минимум **2**, максимум **100** участников.
- Каждый участник имеет: `id`, `name`, `color`, флаги `alive` / `eliminated`, `rank` (место при выбывании).
- Имена генерируются случайно из пула русских имён (включая шутливые ники). Цвета распределяются равномерно по цветовому кругу HSL.

### Условие победы

Игра заканчивается, когда остаётся **один живой участник**. Ему показывается модальное окно «Победитель».

### Выбывание (Elimination)

- При выбывании участник получает ранг: чем раньше выбыл, тем выше номер места (последний выбывший — 2-е место, победитель — 1-е).
- **Кулдаун на выбывание: 10 секунд.** Если за этот период должны выбыть несколько участников, лишние ставятся в очередь и обрабатываются по одному после снятия кулдауна.
- При выбывании воспроизводится звук и визуальный эффект (частицы).

### Эскалация

Если **30 секунд** прошло без единого выбывания, включается режим **эскалации** на 5 секунд:

- Индикатор меняется с «НОРМАЛЬНО» на «ЭСКАЛАЦИЯ».
- Экран мигает, камера трясётся, звучит сигнал.
- Каждая игра реагирует по-своему (подробности — в документации конкретной игры).

Эскалация сбрасывает таймер «тишины» и может повторяться, пока игра не завершится.

### Интерфейс арены

- **Таймер** — время с начала текущей игры.
- **Панель участников** — живые сверху, выбывшие снизу под разделителем «Выбывшие».
- **Индикатор эскалации** — текущий режим темпа игры.

## Запуск локально

Проект не требует установки зависимостей. Достаточно открыть `index.html` в браузере или поднять локальный HTTP-сервер:

```bash
# Python 3
python3 -m http.server 8000

# Node.js (npx)
npx serve .
```

Затем откройте `http://localhost:8000`.

## Добавление новой игры

1. Создайте файл в `games/` с объектом, содержащим поля `name`, `description`, `icon` и методы:
   - `init(canvas, players, callbacks)` — инициализация
   - `update(dt)` — логика кадра (dt в секундах)
   - `draw(ctx)` — отрисовка
   - `onEscalation()` — реакция на эскалацию (опционально)
   - `destroy()` — очистка ресурсов
2. Подключите скрипт в `index.html`.
3. Добавьте игру в массив `games` в `main.js` → `renderGamesGrid()`.
4. Для выбывания вызывайте `callbacks.eliminatePlayer(playerId)`.

## Лицензия и контрибуция

Проект является клиентским веб-приложением без серверной части. Все вычисления выполняются в браузере пользователя.
