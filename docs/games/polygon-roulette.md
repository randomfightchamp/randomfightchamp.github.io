# Арканоид Судьбы 🎯

> **Файл:** `games/polygonRoulette.js`  
> **ID:** `polygon-roulette`  
> **Слоган:** Два вложенных многоугольника, вращающихся в противоположных направлениях. Шарик разрушает внутренний и выбивает внешних игроков.

## Суть игры

На арене два вложенных вращающихся многоугольника. Белый шарик отскакивает от граней внутреннего многоугольника, разрушая их. Когда шарик касается грани внешнего многоугольника — привязанный к ней участник выбывает. Игра идёт раундами до одного победителя.

## Структура арены

### Внутренний многоугольник

| Параметр | Значение |
|----------|----------|
| Радиус | До 210 px; автоматически уменьшается, если внешний многоугольник мал |
| Число граней | 30 (фиксировано) |
| Вращение | По часовой, 0.6 рад/сек |
| Роль | Препятствие для шарика (как кирпичи в арканоиде) |

### Внешний многоугольник

| Параметр | Значение |
|----------|----------|
| Радиус | 350 px |
| Число граней | = числу живых участников |
| Вращение | Против часовой, −0.25 рад/сек |
| Роль | Каждая грань принадлежит одному участнику |

Грани привязаны к стабильному `playerId`, поэтому при выбывании одного участника цвета оставшихся граней не «съезжают» на соседние сектора.

### Финал (2 участника)

Внешний многоугольник становится **квадратом** (4 грани): по **2 грани** на каждого участника.

### Три участника

Когда остаётся 3 участника, внешний многоугольник — треугольник. Внутренний 30-угольник автоматически уменьшается, чтобы помещаться внутри треугольника с запасом.

## Шарик

| Параметр | Значение |
|----------|----------|
| Радиус | 7 px |
| Начальная скорость | 260 единиц |
| Максимальная скорость | 500 единиц |
| След | 20 последних позиций |

Шарик запускается из центра в случайном направлении.

## Фазы игры

| Фаза | Длительность | Описание |
|------|--------------|----------|
| `preparation` | 2 сек | Показ номера раунда и числа оставшихся участников |
| `running` | — | Активная игра |
| `round_pause` | 1.8 сек | Пауза после выбывания |
| `ending` | — | Объявление победителя |

## Механика столкновений

### Шарик × внутренняя грань

1. Шарик **отскакивает** по правилу арканоида: `v' = v − 2(v·n)n`.
2. Скорость нормализуется (макс. 500).
3. Грань **деактивируется** (исчезает).
4. Появляются частицы-осколки.
5. За кадр обрабатывается **не более одной** грани.

### Шарик × внешняя грань

1. Определяется участник, привязанный к грани (`playerId`).
2. Ударенная грань сразу **деактивируется** именно в месте касания.
3. Участник **выбывает**.
4. Показывается имя выбывшего.
5. Если остался не один участник, начинается пауза между раундами.

### Защита от «вылета в космос»

Если шарик улетает дальше `OUTER_R + 50` от центра, принудительно выбывает участник **ближайшего сектора** по углу.

## Раунды

После каждого выбывания:

1. Пауза **1.8 секунды** (отображается номер раунда).
2. Внешний многоугольник **пересоздаётся** с числом граней = числу живых.
3. Шарик **сбрасывается** в центр.
4. Внутренний многоугольник **восстанавливается** (все 30 граней активны).
5. Номер раунда увеличивается.
6. Перед стартом нового раунда снова показывается короткая подготовка, в это время шарик не движется.

## Условие победы

Остаётся **1 участник** — он побеждает. На экране: «🏆 ПОБЕДИТЕЛЬ 🏆» и имя.

## Эскалация

При общей эскалации:

| Эффект | Значение |
|--------|----------|
| Разрушение внутренних граней | До **5 случайных** активных (минимум **8** остаётся) |
| Скорость шарика | **×1.25** (макс. 500) |
| Скорость вращения внутреннего | **×1.5** |
| Визуал | Пульсирующий красный цвет граней (3 сек) |

## Визуальные элементы

- **Внешние грани** — цвет участника, толщина 6 px, свечение.
- **Имена участников** — серые полупрозрачные подписи за внешними гранями, вращаются вместе с секторами.
- **Внутренние грани** — белые (красные при эскалации).
- **Разделители секторов** — тонкие линии от центра.
- **Шлейф шарика** — полупрозрачные следы.
- **Последний выбывший** — баннер сверху и подпись во время паузы между раундами.

## Динамика игры

Каждый раунд гарантированно убирает одного участника (если шарик достигает внешней грани). По мере сокращения числа игроков внешний многоугольник упрощается, а в финале превращается в квадрат с двумя секторами на игрока. Разрушение внутренних граней открывает траектории к внешнему кольцу.
